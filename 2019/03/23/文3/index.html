手机端使用Termux搭建Hexo博客+部署到github

· 2019-3-23 · 次阅读

  • 1.安装termux终端
  • 2.打开termux，出现installing请等待或使用梯子加快速度

  • 3.依次执行命令pkg install coreutils //安装coreutils软件包
    pkg install git //安装git
    pkg install nodejs //安装node.js

  • 4.由于官方npm镜像在国内会卡，所以使用淘宝镜像
    npm config set registry https://registry.npm.taobao.org

  • 5.继续依次执行命令npm install hexo-cli -g //安装hexo
    mkdir hexo
    cd hexo //新建hexo文件夹并打开
    hexo init //初始化hexo

  • 6.termux中生成的文件夹路径
    /data/data/com.termux/files/home/hexo

  • 7.配置Git全局信息，这样才能提交数据
    git config –global user.name “你的Gitee用户名”
    git config –global user.email “你Gitee使用的邮箱”

  • 8.修改配置文件
    进去github查看我们创建博客的仓库，记录下git地址

在数据根目录，找到这个文件_config.yml配置文件打开它

  • 9.修改1——存放数据路径修改:
    设置_config.yml的信息
    uri: 你的github名.gitub.io
    type: git
    末尾加上
    repo: 你的git地址
    branch: master
  • 10.保存退出(个人推荐手机上使用MT管理器)
  • 11.我们来测试一下本地文件能否于gitee端传输数据上去

先安装 hexo-deployer-git插件，在数据目录执行:npm install hexo-deployer-git –save

生成静态文件，并上传至Gitee端，部署到服务器
在数据目录git bash 执行下命令
hexo g && hexo d
查看仓库，我们看到文件有更新，说明我们对 接成功了
这样子就完成了部署
我们就可以直接访问这个网址进入自己的博客了

  • 12.写博客怎么写呢？
    本地博客保存在:
    数据目录\source_posts\md文件
    我们只要写好的博客用Markdown编辑器，导出md文件放入此文件夹，再次执行
    hexo g && hexo d命令
    即可提交至服务器端

关于仓库ssh和github的注册等等
本文就不一一细说了.百度就可以了

还有个小细节你退出终端后第二次怎么使用hexo呢？
输入cd hexo便可！
